# 記帳 APP（安娪專屬）

手機網頁式記帳 App（PWA）。完整規格與決策記錄在 Claude 記憶 `project_accounting_app.md`。

## 檔案
- `index.html` / `app.css` / `app.js` — 主程式（純前端，無框架）
- `manifest.webmanifest` / `sw.js` / `icon-*.png` — PWA 安裝與離線
- 資料存瀏覽器 localStorage，key：`anyu-ledger-v1`

## 已完成（v1，2026-06-12）
- 首頁：支出/收入頁籤、本月卡（已花/預算條/還可花/平均每天）、今天/本月明細
- 記一筆：純圖示色塊一次點擊直達 → 隔頁頁籤面板（換類不丟金額）、金額鍵盤、日期/地點/付費方式 chips、細項
- 預算頁：總預算＋分類預算（同款面板）、固定支出資訊卡
- 菜市場模式：開關後新支出自動帶地點=菜市場、付費=現金
- 收入：給錢的人＋細項＋金額
- 點明細列可編輯/刪除
- 設計規格：粉圓體 letter-spacing .05em、色票（#16897A/#DE3A2D/#F0BD3B/#7A5AA4/#E69AAE/底#F2EBD8）
- 分類圖示＝Material Symbols Rounded 實心填滿（FILL 1），class `.mi`（letter-spacing:normal 才能形成 ligature）。UI 細部圖示仍用 Tabler outline。安娪要求全填滿、不要線條風（2026-06-12）。圖示名在 CATS map 的 `i:` 欄，之後可隨時換、未來可替換成她手繪 SVG。

## 固定支出（2026-06-12，完全自動記入）
- `db.recurring = [{id,cat,amount,detail,pay,day,active}]`。設定頁入口＝預算頁底部「固定支出」卡 → `ui.view='recurring'`。
- `materializeRecurring()` 啟動時跑：每個 active 範本，若本月還沒記入且 today>=當月扣款日（日數超過月底自動 clamp）就自動產生一筆 expense（帶 `recurId`）。
- 未到期的在設定頁顯示「現在記」→ 開記帳面板（帶 recurId、日期可改＝支援提前付款），存檔後該月不再自動產生（靠 recurId 去重）。
- 自動記入的就是普通 expense，可改日期/金額/刪除。刪範本不刪已記入的帳。
- 編輯範本用 sheet kind `recur`（分類 tabs＋金額＋每月N號 chip(pickDay)＋付款 chip＋啟用/暫停 chip＋細項）。
- 安娪選「完全自動記入」（不要一鍵確認）。

## 改版要點（重要）
每次改 app.css / app.js，要同步把 index.html 裡的 `?v=N` 和 sw.js 的 `CACHE`／SHELL `?v=N` 一起 +1，否則瀏覽器和 service worker 會抱著舊版不放（開發時就踩過）。

## 明細列圖示（2026-06-12）
entryRow 的 meta 用 `metaPart(icon,label,color)`：分類＝CATS 圖示染分類色、地點＝`location_on`(Google 地圖 pin)、付款＝`PAY_ICON` map（現金 payments／刷卡 credit_card／匯款 account_balance），皆 muted。`.mi` 預設白色，這裡用 inline `color:` 覆蓋（inline 勝 class）。
